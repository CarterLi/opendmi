== Name

`dmi_memory_array_addr` -- provides the address mapping for a physical
memory array.

== Library

OpenDMI C library (`libopendmi`, `-lopendmi`)

== Synopsis

[source, c]
----
#include <opendmi/entity/memory-array-addr.h>

typedef struct dmi_memory_array_addr {
    uint64_t start_addr;
    uint64_t end_addr;
    uint64_t range_size;

    dmi_handle_t array_handle;
    dmi_entity_t *array;

    unsigned short partition_width;
} dmi_memory_array_addr_t;
----

== Description

This structure provides the address mapping for a physical memory array. One
structure is present for each contiguous address range described.

=== Fields

`start_addr`::
Physical address, in bytes, of a range of memory mapped to the specified
physical memory array.

`end_addr`::
Physical ending address, in bytes, of the last of a range of addresses mapped
to the specified physical memory array.

`range_size`::
Mapped address range size in bytes.

`array_handle`::
Handle associated with the physical memory array to which this address range
is mapped. Multiple address ranges can be mapped to a single physical memory
array.

`array`::
Physical memory array to which this address range is mapped. Multiple address
ranges can be mapped to a single physical memory array. May be `NULL` when the
memory array handle is incorrect (e.g. due to SMBIOS data errors).

`partition_width`::
Number of memory devices that form a single row of memory for the address
partition defined by this structure.

== Examples

[source, c]
```
dmi_entity_t *entity;
dmi_memory_array_addr_t *info;

info = dmi_entity_info(entity, DMI_TYPE_MEMORY_ARRAY_ADDR);

printf("starting address: %llu\n", info->start_addr);
printf("ending address: %llu\n", info->end_addr);
printf("range size: %llu\n", info->range_size);
```

== Standards

DMTF SMBIOS Specification, version 2.1 or newer.

== Specification

[cols="1,1,1,1,3"]
|===
| Offset | Version | Name | Length | Description

| `0x00`
| 2.1+
| Type
| `BYTE`
| Memory Array Mapped Address indicator

| `0x01`
| 2.1+
| Length
| `BYTE`
| Length of the structure, `0x0F` for version 2.1, `0x1F`` for version 2.7 and later.

| `0x02`
| 2.1+
| Handle
| `WORD`
| Handle associated with the structure.

| `0x04`
| 2.1+
| Starting address
|`DWORD`
| Physical address, in kibibytes, of a range of memory mapped to the specified
physical memory array. When the field value is `0xFFFFFFFF`, the actual address
is stored in the extended starting address field. When this field contains a
valid address, ending address must also contain a valid address. When this field
contains `0xFFFFFFFF`, ending address must also contain `0xFFFFFFFF`.

| `0x08`
| 2.1+
| Ending address
| `DWORD`
| Physical ending address of the last kibibyte of a range of addresses mapped
to the specified physical memory array. When the field value is `0xFFFFFFFF`
and the starting address field also contains `0xFFFFFFFF`, the actual address
is stored in the extended ending address field. When this field contains a
valid address, starting address must also contain a valid address.

| `0x0C`
| 2.1+
| Memory array handle
| `WORD`
| Handle associated with the physical memory array to which this address range
is mapped. Multiple address ranges can be mapped to a single physical memory array.

| `0x0E`
| 2.1+
| Partition width
| `BYTE`
| Number of memory devices that form a single row of memory for the address
partition defined by this structure.

| `0x0F`
| 2.7+
| Extended starting address
| `QWORD`
| Varies Physical address, in bytes, of a range of memory mapped to the specified
physical memory array. This field is valid when starting address contains the
value `0xFFFFFFFF`. If starting address contains a value other than `0xFFFFFFFF`,
this field contains zeros. When this field contains a valid address, extended
ending address must also contain a valid address.

| `0x17h`
| 2.7+
| Extended ending address
| `QWORD`
| Physical ending address, in bytes, of the last of a range of addresses mapped
to the specified physical memory array. This field is valid when both starting
address and ending address contain the value `0xFFFFFFFF`. If ending address
contains a value other than `0xFFFFFFFF`, this field contains zeros. When this
field contains a valid address, extended starting address must also contain a
valid address.
|===
